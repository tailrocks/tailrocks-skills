# Handoff and Index

How the `research` skill wires the plans together so the next agent (or you, or a
cheaper executor) can pick up with zero context: the `plans/README.md` index, the
item `README.md` update, and the "Next Agent Prompt" handoff.

## Plans index: `proposals/<slug>/plans/README.md`

Written once after all plans, updated by executors as they work:

```markdown
# Implementation Plans — <slug>

Generated by the research skill on <date> at commit `<short SHA>`. Execute in the
order below unless dependencies say otherwise. Each executor: read the plan fully
before starting, honor its STOP conditions, and update your row when done.

## Execution order & status

| Plan | Title | Priority | Effort | Depends on | Status |
|------|-------|----------|--------|------------|--------|
| 001  | ...   | P1       | S      | —          | TODO   |
| 002  | ...   | P1       | M      | 001        | TODO   |

Status values: TODO | IN PROGRESS | DONE | BLOCKED (one-line reason) | REJECTED
(one-line rationale)

## Dependency notes

- 002 requires 001 because <reason>.

## Open unknowns carried from research

- <unknown scoped out of the plans, and why it is safe to defer>.
```

## Item README update: `proposals/<slug>/README.md`

After planning, update the proposal item the `propose` skill created:

- Flip **Status** `EXPLORING` → `PLANNED`.
- Add a **Chosen direction** section naming the direction and why (one paragraph,
  linking `research/00-summary.md`).
- Link the research chapters and the plans index.
- Rewrite the **Next Agent Prompt** (below).

## Next Agent Prompt

The single most important handoff artifact: the block a fresh agent reads first.
It must state current status, the exact next pickup point, active blockers, and
the global checklist. Keep it at the bottom of the item `README.md`.

```markdown
## Next Agent Prompt

> **Status**: PLANNED — <N> plans ready under `plans/`, none started.
> **Pick up at**: `plans/001-<slug>.md`. Read it fully, run its drift check
> first, honor its STOP conditions.
> **Then**: proceed in the order in `plans/README.md`; update each status row.
> **Blockers**: <none | the input X from plan 002's "Inputs to provide">.
> **Global checklist**:
> - [ ] 001-<slug> — <one line>
> - [ ] 002-<slug> — <one line>
```

## Why this shape

- The **index** gives order, dependencies, and live status at a glance.
- The **Next Agent Prompt** removes the "where do I start?" gap that costs a
  fresh agent its first several tool calls.
- Keeping both in the item folder means the whole story — idea, findings,
  evidence, plans, status — travels together and survives a context reset.
